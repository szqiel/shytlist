# Contributing to Shytlist

First off, thanks for taking the time to contribute! Shytlist is a tool built by filmmakers, for filmmakers, and we welcome any help to make it better.

## How Can I Contribute?

### Reporting Bugs
* Check the [GitHub Issues](https://github.com/SZQIEL/Shytlist/issues) to see if the bug has already been reported.
* If not, open a new issue using the **Bug Report** template.
* Provide as much detail as possible, including steps to reproduce and your environment details.

### Suggesting Enhancements
* Open a new issue using the **Feature Request** template.
* Explain why the feature would be useful and how it fits the "Kinetic Viewfinder" philosophy.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the project's design principles (Noir Cinema aesthetic, spring physics, etc.).
5. Issue that pull request!

## Design Principles

When contributing to the UI/UX, please adhere to our [Design System](DESIGN.md) (if available):
* **The Rarity Rule:** Keep neon accents below 10% of pixels.
* **The Mono Data Rule:** Use Geist Mono for all technical/numerical data.
* **Noir Cinema:** Stick to the deep black (#050505) and high-contrast Cyan/Yellow palette.
